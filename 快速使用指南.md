# 快速使用指南

## 🚀 启动应用

```bash
npm run dev
```

---

## 📋 核心功能

### 1️⃣ 打开工作区文件夹
- 点击 "打开文件夹" 按钮
- 选择您的工作区文件夹
- 如果不是标准 Temu 工作区，会提示初始化

### 2️⃣ 新建产品
- 点击 "新建产品" 按钮（工具栏）
- 填写表单：
  - **产品类型**: CD（卡片）、ST（贴纸）或自定义
  - **刀模**: HasOrigin / NoOrigin
  - **规格**: 6pcs、50pcs 等
  - **英文名**: Candle
  - **中文名**: 蜡烛贴纸
- 点击 "创建"

### 3️⃣ 自动生成

系统会自动：
1. 计算唯一序号（如：CD001、CD002...）
2. 生成文件夹名称（如：`CD001_20251217_NoOrigin_6pcs_Candle_蜡烛贴纸`）
3. 创建 4 个子文件夹
4. 生成 `GoodsInfo.md` 产品信息文档

---

## 📁 标准结构

```
TemuWorkspace/
├─ 00_Assets/              # 公共素材库
├─ 01_In_Progress/         # 选品池
├─ 02_Listing/             # 制作中
│  └─ CD001_20251217_NoOrigin_6pcs_Candle_蜡烛贴纸/
│     ├─ 01_Ref_Images/
│     ├─ 02_Ai_Raw/
│     ├─ 03_AI_Handle/
│     ├─ 04_Final_Goods_Images/
│     └─ GoodsInfo.md
├─ 03_Waiting/             # 已发货
├─ 04_Active/              # 已上架
└─ 05_Archive/             # 已下架
```

---

## 🎯 工作流程

1. **新建产品** → 系统自动创建文件夹
2. **导入参考图** → 放入 `01_Ref_Images/`
3. **AI 生成** → 放入 `02_Ai_Raw/`
4. **ComfyUI 处理** → 放入 `03_AI_Handle/`
5. **最终成品** → 放入 `04_Final_Goods_Images/`
6. **记录信息** → 编辑 `GoodsInfo.md`

---

## ⚙️ 序号规则

- **全局唯一**: 跨所有目录（选品池、制作中、已上架等）
- **按类型计数**: CD001、CD002... / ST001、ST002...
- **自动递增**: 扫描已有产品，自动取最大序号 +1

**示例**:
- 已有: CD001（制作中）、CD003（已上架）
- 新建 CD 类产品 → 自动分配 **CD004**

## 🔧 快捷键

- **F12**: 打开开发者工具
- **Ctrl+R**: 强制刷新
- **Ctrl+O**: 打开文件夹（计划中）

## ❓ 常见问题

**Q: 序号会重复吗？**  
A: 不会。系统会扫描所有目录，确保全局唯一。

**Q: 可以自定义产品类型吗？**  
A: 可以。选择 "自定义类型"，输入缩写（如：BK-书签）。

**Q: 文件夹名称太长怎么办？**  
A: 建议英文名和中文名简短（10 字以内）。

**Q: 可以修改已创建的产品吗？**  
A: 目前需手动重命名文件夹。未来版本会支持编辑功能。
